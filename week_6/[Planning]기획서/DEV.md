# DEV.md - 공감 톡톡 개발 계획

## 1. MVP 범위

3주 안에 반드시 완성할 핵심 기능은 아래 3개다.

| 기능 | MVP 포함 | 이유 |
|---|---:|---|
| 다자 AI 토론 | Yes | 공감 톡톡의 핵심 차별점. 한 안건에 분석가, 전략가, 감성케어 3개 관점을 동시에 보여준다. |
| Action Plan 요약 | Yes | 사용자가 AI 답변을 읽고 끝나는 것이 아니라 실제 미팅 전 실행 목록으로 가져가게 한다. |
| 소통 스타일 가이드 | Yes | 같은 메시지를 직설형, 데이터형, 우회형으로 바꿔 실제 말하기 문장까지 제공한다. |
| 회원가입/로그인 | No | 3주 MVP에서는 진입 장벽을 낮추기 위해 익명 사용으로 시작한다. |
| 결제/구독 | No | 데모데이 이후 유료화 검증 단계에서 도입한다. |
| RAG 성공사례 DB | No | 좋은 기능이지만 MVP의 핵심 검증보다 무겁다. |
| 실시간 협업 | No | 첫 버전은 1인 사용자의 미팅 준비에 집중한다. |

## 2. 기술 스택

| 영역 | 선택 |
|---|---|
| Frontend | `index.html` 단일 파일, React 18 CDN, Tailwind CDN, Babel |
| Backend | Node.js, Express, `server.js` |
| AI 호출 | 다온 AI Proxy (`code.daonms.com`) |
| 데이터 저장 | v1은 브라우저 `localStorage` |
| 음성 입력 | Web Speech API |
| 음성 출력 | ElevenLabs TTS, 시간 여유가 있을 때 적용 |
| 배포 | 다온서버 Docker + Caddy, `ggt.daonms.com` |

## 3. 핵심 화면 흐름

1. 사용자가 공감 톡톡 웹앱에 접속한다.
2. 미팅 또는 협상 안건을 한 줄로 입력한다.
3. "토론 시작" 버튼을 누르면 분석가, 전략가, 감성케어 AI가 동시에 응답한다.
4. 사용자가 "Action Plan 만들기"를 누르면 핵심 목표, 실행 항목, 말하기 문장이 정리된다.
5. 필요한 문장을 복사해 실제 미팅 노트나 메신저에 붙여 넣는다.

## 5. 성공 기준

| 기준 | 목표 |
|---|---|
| 사용자가 첫 화면에서 할 일을 이해하는가 | 10초 안에 안건 입력 가능 |
| 다자 토론 결과가 나오는가 | 3개 페르소나 응답이 한 화면에 표시 |
| 실제로 가져갈 결과가 있는가 | Action Plan과 소통 문장 복사 가능 |
| 데모 가능한가 | 발표 중 1분 안에 핵심 흐름 시연 가능 |

## 6. v1에서 하지 않을 것

- 로그인, 결제, 관리자 화면은 만들지 않는다.
- 완벽한 모바일 앱을 목표로 하지 않는다. 반응형 웹까지만 한다.
- 심리 치료나 법률 자문처럼 보이는 표현은 쓰지 않는다.
- MBTI 모드는 재미 요소로 남기고, 첫 데모의 핵심은 직무 미팅 준비에 둔다.
