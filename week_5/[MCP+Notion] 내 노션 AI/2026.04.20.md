## 배운 것 (핵심 3가지)
1. **LLM tool_use (function calling) 패턴** — LLM이 직접 "어떤 함수를 호출할지" 결정하고, 서버가 실행 후 결과를 다시 LLM에 넘기는 흐름. 단순 챗봇과 완전히 다른 수준의 AI 제어
2. **pg 직접 연결 + DATE timezone 버그** — Supabase anon key 없이 `DATABASE_URL`만으로 pg Pool 직접 연결 가능. `to_char(date, 'YYYY-MM-DD')`로 DATE 타입 timezone 오프셋 버그 해결
3. **Vercel 서버리스 듀얼 모드** — `module.exports = app` + `require.main === module` 패턴으로 Vercel 서버리스 환경과 로컬 `node server.js` 모두 동작하게 구성

## 어려웠던 점
- Telegram 409 Conflict — 봇 인스턴스가 2개 이상 동시에 polling하면 발생. 이전에 실행한 node 프로세스를 `kill`로 정리해서 해결
- Notion `object_not_found` 오류 — DB를 만들어도 Notion 통합(integration)에 연결을 안 하면 API에서 보이지 않음. 노션 페이지 설정에서 통합 추가 후 해결
- Supabase anon key 없이 DB 접근 방법 — Supabase 대시보드에서 `DATABASE_URL`을 확인해 pg Pool로 직접 PostgreSQL에 연결하는 방법으로 해결
- DATE 타입 timezone 오프셋 버그 — PostgreSQL DATE 값을 JS로 가져오면 UTC 기준으로 하루 밀리는 현상. `to_char(date, 'YYYY-MM-DD')`로 문자열 변환해서 해결
- Vercel serverless 환경에서 `express.listen()` 충돌 — `require.main === module` 조건으로 로컬에서만 listen() 실행, Vercel에서는 app 객체만 export하도록 분기

## 코드 하이라이트
```javascript
// server.js — Vercel 서버리스 + 로컬 듀얼 모드 핵심 패턴
const app = express();
// ... 라우터 설정 ...

// Vercel에서는 app을 export, 로컬에서는 직접 실행
module.exports = app;
if (require.main === module) {
  app.listen(3000, () => console.log('로컬 서버 실행: http://localhost:3000'));
}
```
**왜 인상 깊었나**: 파일 하나로 Vercel 서버리스와 로컬 개발 서버를 동시에 지원. 배포 환경에 맞춰 별도 파일을 만들 필요 없이 조건 분기 한 줄로 해결하는 실전 패턴
