# Week 회고 자동화 스킬

AFM 2기 평일반 주차 마감 시 그 주의 학습을 회고합니다.

## 실행 절차

### 1. 주차 데이터 수집
```bash
WEEK=$1
# 그 주 일기 전체
ls week_${WEEK}/diary/*.md 2>/dev/null

# 그 주 git 커밋
git log --since="7 days ago" --oneline

# 프로젝트 폴더 목록
ls -d week_${WEEK}/*/ 2>/dev/null
```

### 2. 일기 내용 통합 분석
- 각 일기 Read
- 공통 주제·반복 언급된 개념 추출
- "어려웠던 점" 섹션 모아서 성장 포인트로 전환
- "다음에 해보고 싶은 것" → 다음 주 목표 시드

### 3. 회고 파일 작성

`week_N/weekly-retro.md`:
```markdown
# 📊 Week N 회고 — [주차 주제]

> 기간: YYYY.MM.DD ~ YYYY.MM.DD
> 작성일: YYYY.MM.DD

## 🎯 이번 주 한 줄 요약
> [그 주를 관통하는 한 문장]

## 📈 성장 기록

### ✅ 달성한 목표
- [목표 1] — [증거: 파일·날짜]
- [목표 2] — [증거]

### ⚡ 새로 배운 것 TOP 5
1. **[개념]** — [일기에서 발췌한 나의 말]
2. **[개념]** — ...

### 🛠️ 만든 결과물
| 순서 | 프로젝트 | 기술 | 완성도 | 폴더 |
|------|---------|------|--------|------|
| 1 | ... | ... | MVP/완성 | ... |

## 🎢 어려웠던 순간 → 극복

| 어려움 | 어떻게 풀었나 | 배운 점 |
|--------|--------------|---------|
| [문제] | [해결법] | [교훈] |

## 💭 느낀 점 (솔직하게)

### 좋았던 점
- [점 1]
- [점 2]

### 아쉬운 점
- [점 1]
- [점 2]

## 🚀 다음 주 (Week N+1) 목표
- [ ] [구체적 목표 1]
- [ ] [구체적 목표 2]
- [ ] [구체적 목표 3]

## 🔗 그 주 결과물 링크
[week_N/projects 내 파일 자동 나열]
```

### 4. timetable.md 갱신
`Week N` 섹션을 실제 결과물 기준으로 업데이트:
- "결과물" 컬럼 실제 파일 경로로
- "핵심 학습" 줄에 회고에서 추출한 키워드

### 5. 한 주 요약 출력

```
## 📊 Week N 회고 완료

### 숫자로 보는 한 주
- 📄 일기: N편
- 🛠️ 프로젝트: N개
- 📝 커밋: N건
- ⭐ 핵심 개념: N개

### 성장 포인트 TOP 3
1. [포인트]
2. [포인트]
3. [포인트]

### 👉 다음 주 핵심 과제
- [과제 1]

📁 회고 파일: week_N/weekly-retro.md
📄 timetable.md 갱신 완료
```

## 작성 원칙

1. **숫자로 증명** — "많이 만들었다" ❌ → "9개 만들었다" ⭕
2. **일기의 내 말 인용** — 자기 표현 그대로 따옴표 처리
3. **다음 주로 연결** — 회고는 미래를 위한 것
4. **부끄러워 말기** — 실수·아쉬움도 성장의 증거

## 금지

- 수강생이 안 한 걸 했다고 쓰기 ❌
- 일반론·상투어 ❌
- 다음 주 목표 없이 과거 정리만 ❌
