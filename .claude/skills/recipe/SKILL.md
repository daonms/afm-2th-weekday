# /recipe - 냉장고 재료 기반 레시피 생성

## 목적

`ingredients/` 폴더의 재료 JSON 파일을 읽어 가능한 레시피를 만들고, 결과를 `recipes/` 폴더에 마크다운 파일로 저장한다.

## 필수 지시

- **"ingredients/ 폴더의 JSON 파일들을 먼저 읽어라"**
- 레시피는 항상 **1인분 기준**, **15분 이내**, **자취생 난이도** 조건을 만족해야 한다.
- 재료 추가/삭제 시 코드를 바꾸지 않고 JSON 파일만 관리한다.

## 고정 경로

- 재료 폴더: `week_4/[Skill] 냉장고 재료 기반 레시피 제작 스킬/ingredients`
- 결과 폴더: `week_4/[Skill] 냉장고 재료 기반 레시피 제작 스킬/recipes`
- 생성 스크립트: `.claude/skills/recipe/scripts/generate_recipe.py`

## 실행 절차

1. 재료 폴더에 `*.json` 파일이 있는지 확인한다.
2. 아래 명령으로 레시피를 생성한다.

```bash
python ".claude/skills/recipe/scripts/generate_recipe.py" \
  --ingredients-dir "week_4/[Skill] 냉장고 재료 기반 레시피 제작 스킬/ingredients" \
  --output-dir "week_4/[Skill] 냉장고 재료 기반 레시피 제작 스킬/recipes"
```

3. 스크립트 출력으로 생성된 마크다운 파일 경로를 확인한다.
4. 사용자에게 생성 파일 경로와 추천 레시피 제목을 간단히 보고한다.

## 재료 JSON 형식

파일당 재료 1개 원칙을 지킨다.

```json
{
  "name": "계란",
  "quantity": "6개",
  "category": "냉장"
}
```

## 운영 원칙

- 재료 추가: `ingredients/<new-ingredient>.json` 파일 1개 생성
- 재료 삭제: 해당 JSON 파일 삭제
- 스킬/스크립트 코드는 재료 파일 변화와 독립적으로 유지

## 김풍 우승 레시피 분석 (요약 지식)

아래 패턴을 레시피 생성 시 우선 참고한다.

### 1) 토달토달

- 핵심: **토마토 산미 + 달걀 고소함 + 해장형 국물**
- 구성 원리: 볶은 향채(마늘/양파) -> 토마토 충분히 익히기 -> 육수 -> 달걀 마무리
- 집용 축약 규칙(1인분/15분): 토마토 1~2개, 달걀 2개, 물/육수 400~500ml

### 2) 나베류 국물 요리

- 핵심: **맑은 육수 + 채소 + 단백질**의 균형
- 구성 원리: 육수 먼저 -> 단단한 채소 -> 고기/버섯 -> 향채 마무리
- 집용 축약 규칙(1인분/15분): 시판 육수 기반, 얇은 고기 사용, 채소 2~3종만 사용

### 3) 후룩국

- 핵심: **카레 국물 + 면(파스타) + 고기 볶음 베이스**
- 구성 원리: 양파/마늘/고기 볶기 -> 물+카레 블록 -> 면을 국물에서 마저 익히기
- 집용 축약 규칙(1인분/15분): 면 삶는 시간을 반으로 줄여 국물에서 마무리

### 4) 갸루상 케이크/풍멜튀 계열

- 핵심: **가공식품/간편 재료 조합으로 비주얼 완성**
- 구성 원리: 베이스(빵/과자) + 크림 + 토핑을 층층이 반복
- 집용 축약 규칙(1인분/10분): 굽기 없이 조립형 디저트 우선

## 생성 우선순위 규칙

1. `ingredients/` JSON에 있는 재료로 만들 수 있으면 김풍 대표 메뉴를 우선 추천한다.
2. 불가능하면 동일 조리 원리(국물형, 볶음형, 면형, 조립형 디저트)로 대체 레시피를 생성한다.
3. 출력 레시피는 항상 `1인분`, `15분 이내`, `자취생 난이도`를 유지한다.
